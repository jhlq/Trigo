Trigo is essentially the same as Go only played on a triangular grid. The default ruleset is a hybrid one where both territory, stones and captures count as points. The KO rule is simple, the state of the board is not allowed to repeat.

Why hybrid rules? Usually Japanese rules are used due to faster counting but Chinese rules are more robust, any disagreement can simply be played out whereas the Japanese ruleset implements special cases that arguably are flawed in the presence of unremovable KO-threats. No doubt it is confusing why a group that can live via KO is considered dead when the other player can (potentially) remove all KO threats before starting the KO, removing all KO-threats costs a lot of points so why is it just dead? The answer is that it is because the hypothetical playout ("I can capture this, therefore it is dead") emulate Chinese rules where the KO-threats can be removed at no cost. Players are used to get points for captures but China has many more players so instead of choosing one lets merge them, give points for everything! A perk is that those long running 1 point KOs at the end of the game mostly vanish.

The project is open source and can be found at: github.com/jhlq/Trigo
